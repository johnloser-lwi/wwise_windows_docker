Enable diffraction

Enable diffraction processing in Spatial Audio for the sound.

Diffraction simulates the acoustic phenomenon of sound bending around obstacles - either acoustic portals or level geometry, in a virtual environment. For sound to be diffracted, the game must define either level geometry or rooms and portals and send it to Wwise Spatial Audio.

When diffraction is enabled for a sound, the following takes place in Wwise Spatial Audio:

  • One or more paths between the emitter and the listener are calculated.
  • A diffraction coefficient (0%-100%) is calculated based on the sum of angles in the path. A straight line path is considered 0% diffraction, and a path that bends 180 degrees or more is considered 100% diffraction.
  • The diffraction coefficient (%) is converted to an attenuation, low-pass and high-pass filter value using the obstruction curve defined in the Wwise project settings. The sound is attenuated and filtered accordingly.
  • The apparent position of the sound relative to the listener is calculated to create ‘virtual’ positions giving the listener the impression that the sound is propagating around a corner or through a portal.

Note: 3D Game Object Viewer enables the user to see what is going on inside of Wwise Spatial Audio, including level geometry, portals, and both the sound’s diffraction paths and resultant virtual positions.

Note: For diffraction to take effect, the Wwise Spatial Audio library must be initialized and the game must send level geometry or rooms and portals to Wwise Spatial Audio.

Default value: false
